Etsy sellers are surrounded by market signals: competitor listings, search changes, social trends, Reddit conversations, and shopping results. The hard part is not finding another dashboard. The hard part is knowing what matters this week. EtsyPulse turns that noisy market into actionable briefs. The seller enters their shop once, agents monitor the web, and only Judge approved actions reach the dashboard.

Here is the judge safe demo flow. I can paste an Etsy shop URL, or click the CaitlynMinimalist demo shop. Demo mode requires no credentials and uses cached Bright Data style fixtures, so the app is reliable during judging. The important part is that the data still moves through the real EtsyPulse orchestration path: bootstrap the shop, start the monitoring loop, and write activity and debug events as the system works.

Behind the dashboard is a typed seven agent pipeline. The Shop Bootstrap Agent builds the seller profile, listings, seed keywords, likely competitors, and baseline positioning. Keyword and SERP, Competitor Watch, and Trend Scout agents collect market signals. Market Pulse normalizes those signals into clear events. Then Brief Delivery formats the output for a busy seller, not a data analyst.

Bright Data is the web data boundary. EtsyPulse includes abstractions for Etsy products, SERP batches, markdown scraping, batch scraping, discovery, TikTok, Reddit, Instagram Reels, and Google Shopping. In demo mode, those methods load deterministic cached evidence. In live mode, the project includes a real Bright Data Web Unlocker path for markdown scraping. The debug panel makes this transparent with tool names, cache or live mode, latency, response summaries, and redacted request shapes.

The Judge Agent is the product filter. It scores actionability, urgency, confidence, novelty, business impact, and evidence quality. Only signals above threshold become seller facing briefs. This brief recommends a personalized jewelry gift refresh, explains why now, and gives concrete next steps: create or feature a gift ready variation, update gift intent keywords, and show personalization and packaging in the hero photo.

EtsyPulse starts with Etsy sellers, but the architecture can expand to other marketplaces, agencies, and ecommerce intelligence workflows. It is production shaped with FastAPI, React, Docker, Render, Vercel, Postgres, Bright Data, LLM providers, and optional OpenClaw coordination. The value is simple: small sellers get an always on market analyst powered by web data, without drowning in raw analytics.
